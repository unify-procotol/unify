# UniRender Component

The `UniRender` component is the core of Unify UI. It automatically renders your data in various layouts with full customization support.

## üéØ Basic Usage

<Tabs items={['Preview', 'Code']}>
<Tab value="Preview">
<UniRenderExample type="basic" />
</Tab>
<Tab value="Code">
```tsx
import { UniRender } from '@unilab/unify-ui';

<UniRender
  entity={{
    name: 'User',
    fields: [
      { name: 'id', type: 'number', required: true },
      { name: 'name', type: 'string', required: true },
      { name: 'email', type: 'string', required: true },
      { name: 'role', type: 'string' },
      { name: 'isActive', type: 'boolean' }
    ]
  }}
  data={[
    { id: 1, name: 'John Doe', email: 'john@example.com', role: 'Admin', isActive: true },
    { id: 2, name: 'Jane Smith', email: 'jane@example.com', role: 'User', isActive: true },
    { id: 3, name: 'Bob Johnson', email: 'bob@example.com', role: 'User', isActive: false }
  ]}
  layout="table"
  config={{
    id: { label: 'ID', width: '60px' },
    name: { label: 'Full Name' },
    email: { label: 'Email Address' },
    role: { label: 'Role' },
    isActive: { 
      label: 'Status',
      render: (value) => (
        <span className={`px-2 py-1 rounded-full text-xs font-semibold ${
          value ? 'bg-green-100 text-green-800' : 'bg-red-100 text-red-800'
        }`}>
          {value ? 'Active' : 'Inactive'}
        </span>
      )
    }
  }}
  generalConfig={{
    showActions: true,
    actions: {
      edit: true,
      delete: true
    }
  }}
/>
```
</Tab>
</Tabs>

## üìã Props Reference

### Required Props

| Prop | Type | Description |
|------|------|-------------|
| `entity` | `Entity` | Entity schema definition |
| `data` | `any[]` | Array of data records |
| `layout` | `LayoutType` | Layout mode: `table`, `card`, `grid`, `list`, `form`, `dashboard` |

### Optional Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `config` | `Record<string, FieldConfig>` | `{}` | Field-specific configuration |
| `generalConfig` | `GeneralConfig` | `{}` | General rendering configuration |
| `loading` | `boolean` | `false` | Loading state |
| `error` | `string \| null` | `null` | Error message |
| `onAdd` | `(record: any) => void` | `undefined` | Add record callback |
| `onEdit` | `(record: any, index: number) => void` | `undefined` | Edit record callback |
| `onDelete` | `(record: any, index: number) => void` | `undefined` | Delete record callback |
| `onSave` | `(record: any, index: number) => void` | `undefined` | Save record callback |
| `className` | `string` | `""` | Additional CSS classes |

## üèóÔ∏è Entity Schema

Define your data structure using the Entity interface:

```tsx
interface Entity {
  name: string;
  fields: EntityField[];
  schema?: JSONSchema;
}

interface EntityField {
  name: string;
  type: string;
  required?: boolean;
  description?: string;
}
```

## üîÑ Event Handling

Handle user interactions with callback functions:

```tsx
const handleAdd = async (record) => {
  try {
    const newRecord = await api.createUser(record);
    setData([...data, newRecord]);
    toast.success('User created successfully');
  } catch (error) {
    toast.error('Failed to create user');
  }
};

const handleEdit = async (record, index) => {
  try {
    const updatedRecord = await api.updateUser(record.id, record);
    const newData = [...data];
    newData[index] = updatedRecord;
    setData(newData);
    toast.success('User updated successfully');
  } catch (error) {
    toast.error('Failed to update user');
  }
};

const handleDelete = async (record, index) => {
  if (confirm('Are you sure you want to delete this user?')) {
    try {
      await api.deleteUser(record.id);
      setData(data.filter((_, i) => i !== index));
      toast.success('User deleted successfully');
    } catch (error) {
      toast.error('Failed to delete user');
    }
  }
};

<UniRender
  entity={userEntity}
  data={data}
  layout="table"
  onAdd={handleAdd}
  onEdit={handleEdit}
  onDelete={handleDelete}
/>
```

## üöÄ Performance Tips

**Optimization Best Practices**:
- Use `React.memo` for custom render functions
- Implement pagination for large datasets
- Use virtual scrolling for 1000+ items
- Debounce search and filter operations

### Virtual Scrolling Example

```tsx
import { FixedSizeList as List } from 'react-window';

const VirtualizedTable = ({ data, entity }) => {
  const Row = ({ index, style }) => (
    <div style={style}>
      <UniRender
        entity={entity}
        data={[data[index]]}
        layout="table"
      />
    </div>
  );

  return (
    <List
      height={600}
      itemCount={data.length}
      itemSize={60}
    >
      {Row}
    </List>
  );
};
```
